# HU-PIGCCT-SYM-198  
## Épica: Navegación, Inicio y Acceso al Sistema PIGCCT  
### Visualizar perfil

---

## DESCRIPCIÓN HISTORIA DE USUARIO

> **Como:** usuario autenticado.  
> **Quiero:** visualizar la información básica de mi perfil en el sistema.  
> **Para:** conocer mis datos registrados y validar que mi información personal esté correcta.

---

## CRITERIOS DE ACEPTACIÓN

### 1. Acceso al perfil de usuario
1.1 El sistema debe permitir a todo usuario autenticado acceder a la opción **“Mi perfil”**.  
1.2 El acceso debe estar disponible desde el menú principal.  
1.3 El sistema no debe permitir el acceso a usuarios no autenticados.

### 2. Información del perfil
2.1 El sistema debe mostrar como mínimo la siguiente información del usuario:
- Nombre completo.
- Correo electrónico.
- Rol asignado.
- Estado de la cuenta.

2.2 La información debe mostrarse en modo solo lectura.

### 3. Visualización clara y organizada
3.1 La información del perfil debe presentarse de forma clara y estructurada.  
3.2 El sistema debe diferenciar claramente los datos personales y los datos del rol.

### 4. Seguridad de la información
4.1 El sistema debe garantizar que el usuario solo pueda visualizar su propia información.  
4.2 No debe ser posible acceder al perfil de otros usuarios por navegación directa.

### 5. Actualización de la información
5.1 Los cambios realizados por administradores sobre el perfil del usuario deben reflejarse automáticamente.  
5.2 El sistema debe mostrar siempre la información actualizada.

### 6. Usabilidad y experiencia de usuario
6.1 La vista de perfil debe ser clara, sencilla y fácil de entender.  
6.2 El diseño debe ser consistente con la identidad visual del sistema PIGCCT.  
6.3 El sistema debe permitir una navegación fluida hacia otros módulos.

---

## Resultado esperado

El usuario puede **visualizar su perfil personal**, consultando de forma clara su información básica, rol y estado de cuenta, facilitando la autogestión y fortaleciendo la transparencia dentro del sistema PIGCCT.
